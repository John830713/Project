//==============================================================================
// UI Layer - Design Overview
//==============================================================================

Purpose:
  Shared UI components for the desktop pet host. Contains the settings dialog
  used by both pet config and module config.

Contents:
  SettingsDialog     - Modal settings window with "Pet" tab (first) and
                       per-module tabs; reads/writes config fields via
                       IFeatureModule::GetValue/SetValue and PetConfig, with
                       scroll support.

Rules:
   1. SettingsDialog opens as a modal window; Pet tab is always tab 0,
      module tabs follow at index 1..N.
   2. Each module's config fields are editable in a dedicated tab. If fields
      exceed visible area, vertical scrollbar appears.
    3. Tab labels are translated via Tr(L"TabLabels", module->GetDisplayName()).
       Each module must have an entry in the [TabLabels] section of its lang file,
       e.g. "AutoKey=按鍵精靈".
    4. On Save: reads all controls → saves PetConfig → calls module->SetValue()
       for each module → ModuleManager::SaveAllConfigs() → ApplyAllConfigs().
       On Cancel: discards changes.
    5. Missing config keys are auto-filled with defaults by
       ConfigManager::EnsureModuleConfig() during LoadModuleConfig().
       Existing keys are never overwritten.
    6. CRITICAL (button preservation): When a module with HasCustomSettings()=true
       is selected, PopulateFields creates a "Manage Actions..." button (ID 1000)
       after the config fields. When clicked, WM_COMMAND handler:
         a) ReadFieldsFromControls() to save current edits
         b) Call mod->OpenCustomSettings(hwnd)
         c) Do NOT DestroyWindow + PopulateFields (that would destroy the Manage
            button itself; the module's OpenCustomSettings may open a modal child
            dialog, and after closing it the Manage button must still exist).
         d) Just leave existing controls intact.
    7. Module config field labels are translated via Tr(L"Common", def.label.c_str())
       by the SettingsDialog, so standard keys like "Enable Module" go in [Common]
       section of the lang files.
